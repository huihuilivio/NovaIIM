# Admin 管理面板 — 需求文档

## 1. 项目背景

NovaIIM 服务端需要一个 Admin HTTP 管理面板用于：
- 运维人员监控服务状态
- 管理员管理用户/群组/消息
- 审计操作日志

Admin 面板以 **RESTful JSON API** 形式提供，前端由独立项目对接（不含前端实现）。

## 2. 设计原则

- **只读优先**：大部分接口为查询，写操作需鉴权 + 审计
- **RBAC 鉴权**：基于 `roles` / `permissions` / `admin_roles` 表
- **分页标准化**：所有列表接口统一分页参数 `page` / `page_size`
- **无状态**：JWT token 鉴权，不依赖 session

## 3. 功能范围

### P0 — 核心（本次实现）

| 模块 | 功能 | 说明 |
|------|------|------|
| **认证** | 管理员登录 | 用户名+密码 → JWT |
| **仪表盘** | 实时指标 | 连接数、在线用户、消息吞吐、uptime |
| **用户管理** | 用户列表/搜索 | 分页 + uid/nickname 模糊搜索 |
| **用户管理** | 用户详情 | 基本信息 + 在线状态 + 设备列表 |
| **用户管理** | 添加用户 | 管理员创建新用户 |
| **用户管理** | 删除用户 | 软删除（status=3）+ 踢下线 |
| **用户管理** | 重置密码 | 管理员重置用户密码 |
| **用户管理** | 封禁/解封 | 修改 users.status |
| **用户管理** | 踢下线 | 关闭用户所有连接 |
| **消息管理** | 消息查询 | 按会话 + 时间范围分页查询 |
| **消息管理** | 撤回消息 | 修改 messages.status = 1 |
| **审计** | 操作日志 | 写操作自动记录到 audit_logs |

### P1 — 扩展（后续迭代）

| 模块 | 功能 |
|------|------|
| 群组管理 | 群列表/解散/踢人 |
| 会话管理 | 会话列表/详情 |
| 角色权限 | RBAC 角色/权限 CRUD |
| 统计报表 | 日活/消息量趋势图数据 |
| 系统配置 | 运行时参数热更新 |

### 不做

- 前端 UI（由独立前端项目对接 API）
- 文件/图片管理
- 推送管理

## 4. 非功能需求

| 项目 | 要求 |
|------|------|
| 性能 | Admin 流量小，1 worker 线程足够 |
| 安全 | JWT 鉴权 + RBAC 权限校验 + 审计日志 |
| 端口 | 独立端口（默认 9091），与 IM TCP 隔离 |
| 响应格式 | 统一 JSON：`{"code": 0, "msg": "ok", "data": {...}}` |
| 错误码 | 0=成功, 1=参数错误, 2=未登录, 3=无权限, 4=资源不存在, 5=内部错误 |
